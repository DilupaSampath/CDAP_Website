ARIMA model

Time series models are used for predicting or forecasting the future behavior of variables. These models account for the fact that data points taken over time may have an internal structure that should be accounted for. As a result, standard regression techniques cannot be applied to time series data and methodology has been developed to decompose the trend, seasonal and cyclical component of the series. Modelling the dynamic path of a variable can improve forecasts since the predictable component of the series can be projected into the future. Time Series is an ordered sequence of values, y1, y2,...,yT of a quantitative characteristic collected over time.

  • Most often, the observations are made at regular time intervals
  • The essential quality of the series is the order of observations according to the time variable.
  • This makes it distinct from common data problems, where there is no natural ordering of the observations.

Time Series Analysis accounts for

  • Understanding the underlying structure that produced the observed data
  • Formulating a model
  • Proceed to forecasting

There are mainly two types of time series of models.

  • Classical Models
  • Probabilistic Model

Classical Models divided into two types.

  • Additive Models
  • Multiplicative Models

Genetic Algorithm

Genetic algorithm is a search heuristic that is based on Charles Darwin’s theory of natural evolution. This algorithm reflects the procedure of natural selection where the fittest individuals are selected for reproduction to produce offspring of the next generation. Simply it chooses individuals from the current population and uses them as parents to produce the children for the next generation. Over succeeding generations, the population evolves towards an optimal solution.

To achieve the optimization of healthcare staff scheduling, we picked genetic algorithm because;

  • Compared to a classical optimizing algorithm, genetic algorithm generates a population of points at each iteration and the best point in a population is generated as the optimal solution while a classical optimizing algorithm generates a single point at each iteration, and sequence of points are generated as an optimal solution.
  • Faster and more efficient as compared to the classical traditional algorithms.
  • Gets an answer to the problem at all times, which improves over the time.
  • Useful when the search space is bulky and when a large number of parameters are involved.

Our GAs driven trough under five key steps

  • Generate an initial population F(0) with n solutions
  • Compute the fitness value u(f) for each individual solution f in the current population F(t)
  • Generate the next population F(t+1), by selecting i best solutions from F(t)
  • Produce offspring by applying the genetic operators to population F(t+1)
  • Repeat from Step 2 until a satisfying solution.The process is continued until x number of generations are explored and the best solution (the solution with minimum fitness value) is selected as the optimal number of nurses for wards.

Iterated Local Search

Iterated Local Search is more effective and quickly explore the optimal solutions and it is able to find a highly accurate optimum by using a few number of iterations. This algorithm is able to keep track on its current state (current solution) and move to neighboring states (neighborhood) with the results of current state. Then it will repeat the same process if the neighboring state get better than the current state until it gets the best solution.

  • Algorithm can return a schedule in few seconds

Application of ILS algorithm

Iterated Local Search algorithm is used to allocate nurses to wards in an efficient way by considering two main factors such as;

  • Number of patients in a ward
  • Number of nurses in the hospital

The ILS starts with an initial solution. We have used two approaches to generate the initial solution: (1) best fit and (2) random fit. In the best fit approach, the nurses are allocated to wards according to the requirement of each ward (number of nurses required by each ward). Wards are considered sequentially (by the ward number), and nurses are allocated starting from first wards, second ward etc. Therefore, the last wards might not get nurses allocated at all. On the other hand, with the random fit approach, the nurses are allocated to wards randomly, without considering any factors. Next, the initial solution goes through a perturbation process to generate a new solution. In the perturbation process, each nurse is selected sequentially and moved from the allocated ward to another ward. Each of this move generates a new solution. Each new solution will be evaluated using the heurist function. We have used the previously mentioned fitness function as the heuristic function for ILS. If the new solution gives a better fitness value, then the old solution is discarded, and new solution is considered as the current solution. 23 The process is continued until x number of iterations are explored and the best Solution (the solution with minimum fitness value) is selected as the number of tourists that can be accommodated for each location.
